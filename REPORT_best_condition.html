1. 背景と問い

WordAntipode はオープンウェイトLLMの内部表現から「意味的に最も遠い単語ペア」を抽出するツール。 そのためにはまずどのモデル・どの層・どのpoolingが日本語の意味を最もよく拾うのかを確かめる必要がある。 本レポートは6モデル × 3層を3種の実験で評価し、最良条件を決定した記録である。

3. 実験1: 対義語240ペアの類似度

yattoke の対義語240ペアの単語ベクトル間コサイン類似度を測定。 最初の仮説:「対義語は意味的に遠いはず → 負のコサイン値になる」。

モデル系統層平均中央値p10p90負/正
open-calm-medium生成mid+0.255+0.296-0.075+0.52539 / 201
open-calm-large生成mid+0.280+0.278+0.024+0.53221 / 219
e5-large埋込final+0.282+0.265+0.065+0.5513 / 237
bge-m3埋込final+0.286+0.280+0.069+0.5487 / 233
ruri-large埋込final+0.447+0.452+0.202+0.6881 / 239

結果: どのモデルも対義語ペアの平均コサインは正。仮説は外れ。 これは「対義語」が意味空間で対極ではなく「同じ意味領域の対比軸」として配置されることを示す （コントラスト学習の特性）。よってこの実験単体ではモデル間の優劣判定に使えない。

4. 実験2: ランダム500ペアとの差分

vocab.ja.1000 からランダムに作った500ペアの平均コサインと、対義語240ペアの平均コサインを比較。 Δ = (対義語平均) − (ランダム平均) が大きいほど、モデルは「対義語＝同じ領域にある関連語」と認識している = 意味カテゴリの理解がある。

#モデル対義語平均ランダム平均Δ ★
1ruri-large+0.447+0.006+0.440
2bge-m3+0.286+0.003+0.283
3open-calm-large+0.280+0.009+0.270
4open-calm-medium+0.255-0.002+0.258

結果: ruri-large が最大のΔ。意味的に近い語（対義語含む）を最も強くまとめている。 ただしこの指標は「対義語が同じカテゴリに属する」という性質を間接的に測っているだけで、 カテゴリ境界そのものの鮮明さは別途検証する必要がある（→ 実験3）。

5. 実験3: 6カテゴリ意味クラスタリング ★決定打

vocab.ja.1000 から意味的にまとまった6カテゴリ (感情50・自然64・動物64・食66・科学52・身体39 語) を抽出し、以下を計測:

  • within: 同じカテゴリ内の語ペアの平均コサイン類似度
  • cross: 異なるカテゴリ間の語ペアの平均コサイン類似度
  • gap = within − cross: 大きいほど「カテゴリ境界が鮮明」 = 意味理解の証拠

6. 結論と本流への反映

最良条件: cl-nagoya/ruri-large + CLS pooling (first) + 最終層 (24/24)

根拠:
  1. 意味クラスタリングスコア gap = +0.227（2位の1.42倍、最下位の2.08倍）
  2. 6カテゴリ（感情・自然・動物・食・科学・身体）すべてで within 最高
  3. 日本語特化コントラスト学習による意味的まとまりの鮮明さ

本流コードへの反映状況（commit済）

registry.py が cl-nagoya/ など埋込LMプレフィクスを自動検出し、 layer_fraction=1.0 / token_aggregation="first" / add_special_tokens=True を強制する。 CLI から --pooling や --layer-fraction を渡しても埋込LMでは無視される設計。

使い方

uv run --with fugashi --with unidic-lite word-antipode rank \
  -i examples/vocab.ja.1000.txt \
  -m cl-nagoya/ruri-large \
  -n 30 \
  -o out/ruri-large-vocab1000/

ruri-large は形態素解析依存（fugashi+unidic-lite）。 本体依存に追加せず uv run --with で一時的に入れる運用。
