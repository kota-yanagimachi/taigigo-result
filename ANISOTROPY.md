# なぜ「対義語」のコサイン類似度はどれも 1 に近いのか

LLMの内部仕組みに詳しくない人向けの解説メモ。観察された現象、その原因、そして我々が今回入れた対策（mean centering, 通称 A）が **意味的に何をしているのか** を整理する。

## 1. 観察した現象

`pairs.ja.misc.txt`（対義語など17組）を `cyberagent/open-calm-medium` で評価:

| pair | cos |
|---|---|
| 男 / 女 | 0.9999 |
| 強 / 弱 | 0.9999 |
| 過去 / 未来 | 0.9999 |
| 白 / 黒 | 0.9999 |
| Ash Tale / Slay the Spire | 0.9659 |

「対義語」も「ほぼ無関係なゲームタイトル同士」も、揃って 0.96〜0.9999。**何を入れてもほぼ1**。

直感的には「これだけ意味が違うのに、なぜ?」と疑問になる。本ドキュメントはその原因を、LLM内部に詳しくない人にも分かるように整理する。

## 2. 「意味ベクトル」とは何だったか（おさらい）

LLM（言語モデル）は内部で各単語を **数百〜数千次元のベクトル** に変換する。「ベクトル」は要するに長い数値配列で、空間内の点だと思って差し支えない。

このベクトル間の距離が「意味の近さ」と対応する、というのが大方の前提。距離の測り方として **コサイン類似度** がよく使われる:

- `+1` に近い → ベクトルの向きが同じ → 意味が近い
- `0` 付近 → 直交 → 無関係
- `-1` に近い → 真反対の方向 → 意味も逆

…と期待されている、というのが教科書的な話。

## 3. なぜ実際は全部 cos ≈ 1 になるのか

原因は **2つ**、どちらも有名な現象。

### 3.1 異方性 (Anisotropy): ベクトルが「狭い円錐」に押し込まれている

Ethayarajh (2019) が示したのは、BERT・GPT-2・ELMo など主要LMの隠れ状態は **空間全体に均一に分布しておらず、特定の方向（円錐）に集中している** ということ。OpenCALM（GPT-NeoX 系）も同類。

イメージ:

```
理想（均等）:                        現実（異方性）:

        ↑                                    ↑
        ⊕                                   /⊕
   ⊕    │    ⊕                            /⊕⊕⊕
   ──── ● ────         周囲360°に        /⊕⊕⊕⊕⊕    ある1方向に
   ⊕    │    ⊕         満遍なく           /              全部偏っている
        ⊕               散らばっている    ●原点
        ↓
```

円錐内部に全ベクトルが押し込まれているので、どの2点を取っても **方向はほぼ同じ** → cos ≈ 1。これは「単語の意味が似ているから」ではなく **LLMの構造上の癖** によるもの。

我々のデータでもこの現象が確認できる。open-calm-medium で506語をベクトル化し、ランダムに2000ペア取って cos を測ると:

| | mean | std | min | max |
|---|---|---|---|---|
| ランダムペアの cos | **+0.80** | 0.21 | +0.50 | +0.9998 |

「無関係な語ペア」でも平均0.80、最低でも0.50。つまり **どんなペアでも構造的に下駄を履いている**。

### 3.2 Rogue Dimensions: 数次元が類似度を独占している

Timkey & van Schijndel (2021) "All Bark and No Bite" はもっと踏み込んで原因を解剖した。1024次元のベクトルだといっても、**実際に cos の値を決めているのは1〜3個の支配的な次元だけ**。残りの1021次元は cos の数値にほぼ寄与していない。

例えるなら:

- 「身長 + 体重 + IQ + 好きな食べ物 + …」と多項目のプロフィールを比較しているように見えて、
- 実際は **身長だけで判定** していて、しかもみんな似たような身長 → 「全員そっくりですね」と言われている、

…という状態。プロフィールの多様性が、ただ1つの強烈な軸に押しつぶされている。

## 4. 対策(A): 平均ベクトルを引く（mean centering）

今回入れた最小コストの対策は、

> **「全単語ベクトルの平均」を計算し、各ベクトルからそれを引く**

これだけ。実装は `src/word_antipode/metrics.py` の `center_vectors()`。

> **2026-05 時点**: この対策は `rank` / `pairs` / `explore` のいずれでも
> CLI フラグ `--center / --no-center` で **デフォルト on** になっている。
> `--no-center` を明示すると本ドキュメント前半で示した「生のコサイン」状態（cos ≈ 1 に張り付く）に戻る。
> `explore` は PCA が層深度方向に支配されないよう、**層ごと**に同じ処理（per-layer mean removal + 再 L2 正規化）を行う点だけ異なる。

### 4.1 幾何学的な意味

異方性で雲が片側に偏っている → 平均は雲の中心 → これを引けば **雲の中心が原点に移動する**。すると個々のベクトルは「中心からどっち向きにズレているか」を表すようになり、向きの違いが正直に出る。

```
before:                          after centering:

        ↑                                  ↑
       /⊕                             ⊕    │    ⊕
      /⊕⊕⊕                                 │
     /⊕⊕⊕⊕⊕                          ──── ● ────
    /                                       │
   ●原点                              ⊕    │    ⊕
                                            ↓
```

### 4.2 意味的な解釈 — これはどう「意味」を変えるのか

A の操作は、

> **「全単語に共通する成分」を “意味のないバックグラウンド” とみなして取り除く**

ことに相当する。LLMの内部状態には、

- どの単語にも乗っている **「LLM共通のバイアス」**（例: 「自分は日本語LMだ」「これは何かのトークン入力だ」など、その語固有の意味とは無関係な成分）
- 単語固有の **「その語らしさ」** 成分

の **両方** が混ざっている。平均は前者を強く反映するので、それを引けば後者だけが残る。

これは統計でいう **共通効果の除去**（ANOVA の `grand mean` を引くのと同じ発想）。コーパス全体に乗っている DC バイアスを抜くことで、「個別の差」が見えるようになる。

数学的にも示唆的で、「中心化後のコサイン」は **ピアソン相関係数** と一致する。つまり A 適用後の cos は、「全体トレンドを除いた純粋な相関」と読み替えてよい。

### 4.3 実測結果

A 適用前後の、ランダム語ペアの cos 分布:

| | mean | std | p05 | p95 |
|---|---|---|---|---|
| raw | +0.80 | 0.21 | +0.54 | +0.9998 |
| centered | **+0.11** | **0.96** | **-0.96** | **+0.998** |

→ 期待通り、ランダムペアの平均は0付近、std は 4.5倍に拡大、分布は[-1, +1] のほぼ全域へ。**空間が「使える」状態に戻った**。

## 5. ところが対義語ペアは依然として cos ≈ 1

ここからが本題の発見。A を適用しても、

| pair | raw | centered |
|---|---|---|
| 男 / 女 | 0.9999 | 0.9984 |
| 強 / 弱 | 0.9999 | 0.9975 |
| 過去 / 未来 | 0.9999 | 0.9955 |

対義語ペアは依然として cos ≈ 0.99。これは **A が効いていない** のではなく（ランダムペアは0.11まで落ちている）、

> **対義語そのものが LLM 表現の中では「近い」**

という事実が、異方性の霧が晴れて初めて見えるようになった、と解釈すべき。

### なぜ対義語が近いのか

これは distributional hypothesis（分布仮説）の有名な弱点。

- 「男」と「女」は **同じような文脈** に登場する（"〇〇のトイレ"、"〇〇の友達"、"〇〇は強い"…）
- LLM は「同じ文脈に出る単語は似ている」と学習する
- 結果、対義語同士が **非常に近い場所** に置かれる

別の言い方をすれば、LLMの表現空間では「対義」は **遠さ** ではなく、**同じ意味軸上の符号違い** として表れる。たとえば「男 − 女」というベクトル差が「性別」軸の方向を指す。両端点同士のコサインは大きいまま（向きは似ている）でも、空間内に「対義性」を表す軸はちゃんと存在している、という構造。

word2vec の "King − Man + Woman ≈ Queen" 類比が成り立つのも同じ理屈。King と Queen は距離としては近いが、両者を結ぶ「性別」軸が一貫した方向を向いているため類比演算が機能する。

### このプロジェクトへの含意

- `pairs` サブコマンド: 「対義語の cos」自体は意味のある観測になる。高ければ高いほど「LLMは対義語を分離できていない」と読める
- `rank` サブコマンド: 「最も cos が小さいペア = 最も意味が遠いペア」は、対義語ではなく **無関係なペア**（カテゴリの異なる語同士、例: 食べ物 vs 国名）になる可能性が高い
- 「対義語を自動抽出したい」のであれば、コサイン距離以外の手段（軸への射影、対比学習、シソーラス併用など）が必要 — 仕様の再検討マター

## 6. 次にできること

- **B (ABTT)**: 平均だけでなく上位主成分も引く。rogue dimensions を直接攻撃する強化版。Mu & Viswanath (2018) の "All-but-the-Top" が原典
- **C: 使用層の変更**: `--layer-fraction` を変えてスイープ。深い層ほど異方性が強い傾向があり、中間〜下層の方が分離の良い空間を返すことがある
- **指標の再設計**: 「意味的に遠いペア」と「対義語」を別の指標で定義し直す（例: 平均的な「対義方向」軸を学習データから推定し、その軸への投影で評価）
- **explore モード**: `word-antipode explore` を使うと、少数語の隠れ状態が層ごとにどう動くかを直接見られる。PC#1 が「LLM 内部処理の進行方向」に支配されやすいという観察から、軌跡可視化のデフォルト軸を PC#2 × PC#3 にしてある（詳細は `docs/EXPLORE_FINDINGS.md`）。ここでも anisotropy 対策として **層ごと** の中心化を既定 on にしている
