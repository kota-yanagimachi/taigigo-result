# 完成基準 (KPI / 受け入れ基準)

仕様書 §9「完成の定義」を測定可能な形に落とし込んだもの。
※ 実装段階で CLI 名称は `word-antipode`、副次サブコマンドは `pairs`、
さらに探索サブコマンド `explore` を追加した。本表もそれに合わせている。

| # | KPI | 計測方法 | 合格条件 |
|---|-----|----------|----------|
| K1 | 単一CLI体系 | `word-antipode --help` を実行 | rank / pairs / explore の各サブコマンドが表示される |
| K2 | 任意語彙入力 | 100語/2000語の2種類で `rank` 実行 | エラーなく結果ファイルが生成 |
| K3 | 上位N指定 | `--top 5` と `--top 20` を実行 | 両方成功、上位5ペアは同一(順序込み) |
| K4 | N増による追加負荷ゼロ | 同じ vocab で `--top 5` と `--top 100` の実行時間差 | 差が候補集合生成済みなら数秒以内(=本質的にO(N)出力のみ) |
| K5 | 4章仕様反映 | コード検査 | 全層60%層・最終トークン・L2正規化・コサイン類似度が adapter / vectorizer に明示 |
| K6 | 単一/多トークン共存 | 「愛」と「東京特許許可局」を含むファイルで実行 | エラーなく両方の距離が出力される |
| K7 | 表示指標 | HTML/CSV出力を確認 | `cosine`（生のコサイン類似度、-1..+1）が表示されている |
| K8 | 図示 | HTML を確認 | バーまたは分布図がランキング各行 or 全体に表示 |
| K9 | HTML 単独閲覧 | ブラウザで `out/*.html` を開く | ローカルサーバ不要で表示できる |
| K10 | 機械可読データ | `out/*.csv` または `*.json` | 同実行で併出される（`explore` は HTML のみ・対象外） |
| K11 | 副次機能 | `word-antipode pairs ...` | ペアごと cosine が表形式と図で出力（類似度降順） |
| K12 | キャッシュ | 同じ語彙で2回目実行 | 「キャッシュヒット N件 / 新規 0件」のログ + 推論時間が大幅短縮 |
| K13 | キャッシュ無効化 | `--layer-fraction 0.5` のように設定を変えて再実行 | 新規計算が走る(キャッシュキーに layer 含むため) |
| K14 | アダプタ拡張性 | `src/word_antipode/models/registry.py` に新規追加点が明確 | 1ファイル追加 + registry に1行で別モデルが使える事をドキュメントで示す |
| K15 | anisotropy 対策 | `--center` 既定 on で `pairs` を実行 | 「中心化: 平均ベクトルを除去」のログが出る・`docs/ANISOTROPY.md` の挙動と整合 |
| K16 | 探索機能 (`explore`) | 11語で実行し HTML をブラウザ確認 | A. PCA軌跡（5主成分から軸切替）と B. K-gon レーダー（軸ホバーで参照語彙の正/負 top5 表示）が動く |
| K17 | チェックポイント | 大規模語彙のベクトル化を途中 Ctrl-C → 再実行 | キャッシュ済みは推論されず、未処理だけ続きから走る |

## 性能の目安(参考)

| 指標 | 目安 |
|------|------|
| 100語ベクトル化 | M1 MPS で 1分以内 |
| 2000語ベクトル化 | M1 MPS で 5〜10分以内 |
| 上位N探索(候補5万ペア) | 数秒以内 |

数値は実測後に追記する。
