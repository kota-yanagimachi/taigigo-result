1. 目的

1.1 問題定義

L2 正規化済みの単位ベクトル集合 X = {x₁, …, xₙ} ⊂ Sd−1 から、コサイン類似度 xᵢ · xⱼ が最小となるペアの上位 K 個を列挙する。 単位球面上の点の場合、コサイン最小は L2 距離最大と同値なので、これは古典的な「Furthest Pair / 直径」問題に帰着する。

‖xi−xj‖2 = 2(1−xi·xj) ⟹ cos 最小⇔L2 最遠

1.2 WordAntipode における用途と要件

WordAntipode はオープンウェイト LLM の意味ベクトル空間から「意味的に最も遠い単語ペア」を抽出するツール。 語彙数 n は 1,000〜100,000、埋め込み次元 d は 768〜4,096 程度。 厳密な総当たり O(n² d) は n ≥ 5,000 でユーザーの応答性を損なうため、近似アルゴリズムが必須。

  • 精度要件: Recall@2000 ≥ 0.95 (Recall@100 ≥ 0.95)
  • 速度要件: n = 10,000 で 30 秒以内、n = 50,000 で 5 分以内
  • モデル独立性: 新規モデル (生成 LLM, sentence embedding 等) でも同じパラメータで動作

1.3 何を解決するのか / 何は解決しないのか

解決対象: 「与えられたベクトル集合の中で cos が最小のペア top-K を高い recall で列挙する」

非解決対象: 「ベクトル化条件 (層位置, pooling 種別, 中心化の要否) の選択」「単語の意味的最遠性をモデルの埋め込みがどれだけ正確に反映するか」

例えば「カエル」「集中」が cos = −0.31 で最遠ペアとして出てきた場合、本アルゴリズムが返す結果は そのベクトル化条件下では確かに最遠 である。
「カエルと集中は人間の感覚で本当に最も離れた語対か」という問いはモデル選択・層選択・pooling 選択の問題であり、本アルゴリズムの守備範囲外。

2. 既存研究と数学的背景

2.1 問題の同値変換: コサイン最遠 ≡ L2 直径

上述の通り、L2 正規化済みなら 2 ノルムとコサインは相互変換可能。 したがって本問題は 「高次元ユークリッド空間における Furthest Pair (Diameter) 問題」 として扱える。 この同値性により、計算幾何学・ANN 文献の蓄積を直接利用できる。

2.2 Williams 2018: 厳密計算の下限

Williams 2018 (SODA) [arXiv:1709.05282] は 高次元 (d = ω((log log n)²)) における ℓ₂ Furthest Pair の厳密計算は OVC/SETH 仮定下で n2−o(1) 時間を要する ことを証明した。

つまり 厳密解を O(n²) より速く解くアルゴリズムは存在し得ない (仮定下)。我々のケースでは n = 1,000、d = 1,024 なので d ≫ (log log n)² ≈ 10 を満たし、この下限が適用される。

興味深い対比:

問題距離計量厳密計算量
Closest Pairℓ₂2O(d) · n logO(1) n (n に線形)
Furthest Pairℓ₁ / ℓ∞near-linear
Furthest Pairℓ₂essentially n2−o(1) (∞ に近い)

つまりコサイン (= ℓ₂) 最遠ペア問題は厳密には不可避にコストがかかる。これが本研究で 近似アルゴリズムを採用せざるを得ない 根拠。

2.3 Pagh 2016 / Indyk 2003: Approximate Furthest Neighbor

Indyk 2003 が原型を提示し、Pagh ら 2016 が改良した Approximate Furthest Neighbor (AFN) アルゴリズム [arXiv:1611.07303] は、Williams の hardness を回避できる。

主アイデア (Pagh 2016 Algorithm 1):

  1. 前処理: ℓ 個のガウスランダム方向 a₁, …, aℓ で全点を射影。各方向につき射影値上位 m 点を保存。
  2. クエリ: 点 q について各方向で aᵢ · x − aᵢ · q (= q から離れている度合い) の優先度キューを構築。優先度順に候補を取り出し、実コサインを計算。最大 m 件展開で打ち切り。

パラメータの自動導出 (Theorem 1):

ℓ= ⌈2n1/c2⌉ , m= ⌈1+e2ℓ logc2/2−1/3 n⌉

ここで c > 1 は 目標近似比。返される距離は真の最遠の 1/c 倍以上が高確率で保証される。 クエリ時間は Õ(n1/c² · (d + log n)) で sublinear in n。

2.4 PCA 候補プール (本プロジェクト旧手法) の数学的位置付け

旧来の WordAntipode は PCA の上位 nc 軸を取り、各軸の両極 K 個ずつを総組み合わせして候補プールとしていた。 これは Pagh Algorithm 2 の クエリ独立版 の変種:

  • Pagh Alg.2: ランダム方向 × 1 extreme/方向
  • WordAntipode 旧: PCA方向 × 両極 K extremes/方向

PCA はデータ適応的なので変分散方向をうまく取れる利点があるが、(c, δ)-近似の理論保証は失う。 また両極 K 拡張は候補プールサイズを K² nc に膨らませる代わりに各軸の貢献を厚くしている。

2.5 SVD 分解による「PC1 寄与」の演繹的解析

SVD X = UΣVᵀ による PCA 基底展開: xᵢ = Σₖ pᵢₖ vₖ ただし pᵢₖ ~ 𝒩(0, rₖ) (近似)、rₖ は説明分散比。

xi·xj = ∑k pik pjk

PC1 極ペア (i*, j*) (PC1 で正端と負端) の内積 ≈ −r₁ · 2 log n (極値理論)、 他軸寄与は i*, j* が PC1 でのみ選ばれたため 期待値 0、標準偏差 σnoise = √(Σk≥2 rₖ²)。

信号/雑音比:

SNR= r1logn ∑k≥2 rk2

SNR が大きいなら PC1 単独で十分。小さいなら多軸必要。この演繹的命題は実測 (§4) と整合する。

3.2 共通条件

  • 語彙: src/word_antipode/data/reference_vocab.ja.1000.txt (1,000 語)
  • 前処理: L2 正規化 → center_vectors (anisotropy 対策で平均除去 + 再 L2 正規化)
  • 埋め込み層: 生成 LLM は layer_fraction=0.5 + max-abs pooling、encoder は最終層 + CLS pooling (registry が自動分岐)

3.3 Ground Truth の構築

全 C(1000, 2) = 499,500 ペアについて exact cosine を計算し、cos 昇順上位 2,000 ペアを GT とした。 実装は scripts/exp_candidate_pool.py の exact_top_n_chunked。

3.4 候補プール構築の 4 方式

  1. PCA n=1, n=4, n=10: PCA 上位 n 軸の両極 K 個ずつから候補ペア。K = ⌈√(cp_target / n)⌉
  2. Random projection L=4, 10, 20: ランダム方向 L 本で同じく両極 K 個ずつ (Pagh Alg.2 単純化版)
  3. Hybrid: PCA + Random の和集合
  4. Pagh per-point AFN (本提案): 各クエリ点 q について Pagh Alg.1 を実行し、(q, AFN(q)) を集めて top-K へ集約

3.5 評価指標

  • Recall@K: GT 上位 K のうち候補集合の上位 K に何個含まれるか (K = 100, 2,000)
  • NDCG@K: 順位重み付け一致度 (上位ほど重い)

3.6 走査範囲

cp_target (候補プール目標値) = 2,000, 10,000, 50,000, 200,000 ペアで方式間比較。 Pagh per-point は derive_params で c=1.1, 1.5, 2.0 を試行。 全実験は scripts/exp_candidate_methods.py と scripts/exp_pagh_per_point.py に再現性ある形で記録。

閾値 ~9 は経験的だが、SNR の 形 は SVD 解析から演繹的に得られている。

5. 考察

5.1 「Query-adaptive」性が候補プール法に対する決定的優位

PCA 候補プール (旧手法) は全クエリで同じ候補集合を使う query-blind 戦略。 対して Pagh per-point は 各クエリ点ごとに反対側候補を個別に選ぶ query-adaptive 戦略。 候補数を同じくらいに揃えても、後者の方が「q から本当に遠い候補」を厳選できる。 これが §4.5 の劇的な性能差の本質。

5.2 PCA が open-calm-medium で例外的に強い理由

open-calm-medium の PC1 説明分散は 17.1% と圧倒的。SVD 分解 (§2.5) からの帰結として PC1 軸の両極ペアが GT のほぼ全てを占める。 そのため PCA n=1 で 1.000 (perfect) を達成可能。

一方 ruri/e5 (sentence embedding) は PC1=5.5% で PCs がフラット。 PC1 単独では信号が弱く、SNR が 8-10 と低いため PCA 多軸が必要だが、それでも Pagh per-point に大きく劣る。

5.3 Williams 2018 の hardness と本実装の (近似での) 回避

§2.2 の通り、厳密な最遠ペア計算は O(n²) 未満不可。 本実装は近似比 c を導入することでこの hardness barrier を超え、Õ(n1+1/c²) 程度で動作する。 実用上は c=1.5 で n1.44、c=2.0 で n1.25 程度。

5.4 c (近似比) の選び方の指針

c 値意味用途
1.0厳密 (総当たり)n ≤ 2,000 では自動的にこちら
1.2近似 (高品質保証)厳密に近い結果が欲しい場合
1.5既定 (バランス)5 モデルで R@2000 ≥ 0.95 を実測
2.0粗 (高速)大規模語彙でラフな最遠ペアでよい場合
2.5+非常に粗探索ヒューリスティック用途

6. 結論

6.1 推奨アルゴリズム: Pagh per-point AFN

WordAntipode は今後 Pagh 2016 Approximate Furthest Neighbor (per-point 適用) を上位 N 最遠ペア探索の本流 として採用する。 候補プールサイズや PCA 軸数のような データ依存パラメータ はユーザーから隠蔽し、 「近似精度 c」という単一のモデル独立パラメータのみを露出する。

6.2.6 EXACT_THRESHOLD (= 2,000)

n ≤ 2000 の場合は precision によらず常に総当たり (method="exact") で実行。 理由: 小規模では NumPy/BLAS による X @ X.T が高速で、AFN の前処理オーバーヘッドを上回るため。

6.3 残された課題

  • top-K furthest pair 専用の理論研究: Pagh は単一クエリ AFN を提供するのみで、top-K pair 列挙への直接の (c, δ)-近似保証はない。Eppstein et al. 1992 の k 番目最遠ペアを高次元に拡張する研究が望まれる。
  • 1B+ 語彙へのスケール: 本実装は naive vectorize 版で n = 50,000 程度まで実用的。n = 10⁶ 以上では disk-friendly な priority queue が必要。
  • 近似 (c=1.5) のモデル横断の最悪ケース保証: 経験的に 4/5 で R@2000 ≥ 0.98 だが open-calm-medium だけ 0.857。多モデルでのストレステストが必要。

7. 付録

A. Pagh 2016 Algorithm 1 (Query-Dependent) 擬似コード 
// 前処理
generate ℓ Gaussian random unit vectors a_1, ..., a_ℓ ∈ ℝ^d
for i in 1..ℓ:
    project: proj_i[x] := a_i · x for all x ∈ S
    S_i := top-m points by proj_i  // 上位 m 点を保存
    sort S_i by proj_i descending

// クエリ q ∈ ℝ^d
initialize priority queue PQ
for i in 1..ℓ:
    take first element x of S_i (with iterator cursor[i] = 0)
    push (a_i · x − a_i · q, x) into PQ

best ← null
for j in 1..m:
    pop top (priority, x) from PQ
    if best is null or dist(x, q) > dist(best, q):
        best ← x
    advance cursor[i], push next element from S_i if any

return best

近似比保証: dist(q, best) ≥ dist(q, true_furthest) / c with probability ≥ 0.72。

B. SVD による SNR 公式の導出

SVD X = UΣVᵀ。第 k 主成分軸 vₖ への射影 pᵢₖ = xᵢ · vₖ は分散 rₖ = σₖ² / Σₗ σₗ²。

PC1 で正端と負端のペア (i*, j*) について:

  • PC1 寄与: 極値理論より 𝔼[pi*1 · pj*1] ≈ −r₁ · 2 log n
  • PC k≥2 寄与: i*, j* が PC1 で選ばれた条件下では pi*k, pj*k は独立に近く、積の期待値 0、分散 rₖ²

他軸寄与の合計分散: Var ≈ Σk≥2 rₖ²。標準偏差 σnoise = √(Σk≥2 rₖ²)。

SNR= |signal| σnoise = r1·2logn ∑k≥2 rk2

(慣例的に係数 2 を落として簡略化することがある)
